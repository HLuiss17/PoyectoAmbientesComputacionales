Domina GitHub: Guía Esencial de Repositorios y Versiones

Paso 1: Crear un repositorio en GitHub

Sigue estos pasos claros para configurar tu nuevo espacio de código en la nube:

1. Inicia sesión

Accede a tu cuenta de GitHub.

2. Crea un nuevo repositorio

Haz clic en el botón “New repository” o “Nuevo repositorio”, generalmente ubicado en la esquina superior derecha.

3. Configura los detalles del repositorio

Rellena la siguiente información esencial para tu nuevo repositorio:

Nombre: PoyectoAmbientesComputacionales

Un nombre claro para tu proyecto.

Descripción: "Aplicación web, ejecutada en un contenedor Docker."

Describe brevemente de qué trata tu proyecto.

Visibilidad: Pública

Permite que sea visible para todos.

Opciones iniciales:
  • Agregar archivo README (documentación básica)
  • Agregar .gitignore (plantilla Node) (ignora archivos innecesarios)
4. Finaliza la creación

Haz clic en el botón Crear repositorio para completar el proceso.

Paso 2: Clonar el repositorio en tu máquina

  • Copia la URL del repositorio: 
    https://github.com/HLuiss17/PoyectoAmbientesComputacionales.git
  • Abre tu terminal o Git Bash.
  • Ve a la carpeta donde guardarás el proyecto: 
    cd ruta/donde/quieres/guardar
  • Clona el repositorio: 
    git clone https://github.com/HLuiss17/PoyectoAmbientesComputacionales.git
  • Entra a la carpeta: 
    cd PoyectoAmbientesComputacionales
  • Verifica los archivos: 
    ls

Subir Primer Avance a GitHub

Paso 3: Inicializar Git

git init

Paso 4: Verificar conexión al repositorio remoto 

git remote -v

Si no aparece nada, conectar manualmente:

git remote add origin https://github.com/HLuiss17/PoyectoAmbientesComputacionales.git

Paso 5: Verificar el estado 

git status

Paso 6: Crear nueva rama 

git branch devdesarrolladores
git checkout devdesarrolladores

Paso 7: Agregar archivos 

git add .

Paso 8: Realizar el commit 

git commit -m "Primer avance del proyecto"

Paso 9: Subir los cambios a GitHub

Si estás en la rama devdesarrolladores:

git push -u origin devdesarrolladores

Si estás en la rama main:

git push -u origin main

Paso 10: Ver historial de commits 

git log

Paso 11: Obtener todos los cambios

git pull
