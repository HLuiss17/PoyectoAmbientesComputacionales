¿Qué es Git y para qué sirve?

Git es un sistema de control de versiones distribuido que permite gestionar el historial de cambios en los archivos de un proyecto. Es una herramienta fundamental en el desarrollo de software porque ayuda a llevar un control organizado y seguro del código fuente.

Entre sus principales usos se destacan:

  • Guardar versiones del proyecto mediante commits.
  • Colaborar con otros desarrolladores sin sobrescribir el trabajo de los demás.
  • Comparar versiones anteriores y restaurarlas si es necesario.
  • Trabajar en distintas funcionalidades en paralelo mediante ramas (branches).

Git se utiliza frecuentemente junto con plataformas como GitHub, donde se pueden alojar los repositorios de forma remota y colaborar en línea.

¿Qué es Git Bash?

Git Bash es una terminal para sistemas Windows que permite usar comandos de Git y de navegación estilo Unix. Es muy útil para trabajar cómodamente con Git desde la línea de comandos.

  • cd Cambiar de directorio.
  • ls Listar archivos.
  • mkdir Crear carpetas.
  • git clone Clonar un repositorio.
  • git add, git commit, git push Guardar y subir cambios.

Comandos básicos de Git

Comando Descripción
git init Inicializa un nuevo repositorio Git en el directorio actual.
git clone <url> Clona un repositorio remoto (de GitHub, por ejemplo).
git status Muestra el estado actual de los archivos en el repositorio.
git add . Añade todos los archivos modificados al área de preparación (staging).
git commit -m "mensaje" Registra los cambios con un mensaje descriptivo.
git remote add origin <url> Conecta el repositorio local con un repositorio remoto (GitHub).
git push -u origin main Envía los cambios locales a la rama principal del repositorio remoto.
git checkout -b nombre-rama Crea y cambia a una nueva rama.
git branch Lista las ramas locales existentes.
git checkout nombre-rama Cambia de rama.
git pull origin main Descarga y fusiona los cambios del repositorio remoto.
git push origin nombre-rama Sube los cambios a la rama especificada en el repositorio remoto.
git stash Guarda temporalmente cambios no confirmados.
git stash pop Recupera los cambios guardados con stash.
git log Muestra el historial de commits.
git reset HEAD~1 Elimina el último commit (mantiene los cambios en el área de trabajo).
git reset --hard HEAD~1 Elimina el último commit y borra los cambios del área de trabajo.
