¿Qué es Git y para qué sirve?

Git es un sistema de control de versiones distribuido que permite gestionar el historial de cambios en los archivos de un proyecto. Es una herramienta fundamental en el desarrollo de software porque ayuda a llevar un control organizado y seguro del código fuente.

Entre sus principales usos se destacan:

  • Guardar versiones del proyecto en diferentes momentos mediante commits.
  • Colaborar con otros desarrolladores sin sobrescribir el trabajo de los demás.
  • Comparar versiones anteriores y restaurarlas si es necesario.
  • Trabajar en distintas funcionalidades de manera paralela mediante ramas (branches).

Git es muy utilizado junto con plataformas como GitHub, donde se pueden alojar los repositorios de forma remota y colaborar en línea.

¿Qué es Git Bash?

Git Bash es una terminal que permite usar los comandos de Git en sistemas operativos Windows. Emula un entorno de línea de comandos de tipo Unix (como en Linux o macOS), facilitando la ejecución de instrucciones.

Git Bash permite usar tanto comandos de Git como comandos básicos de navegación y manipulación de archivos. Algunos ejemplos comunes:

  • cd Cambiar de directorio.
  • ls Listar archivos.
  • mkdir Crear carpetas.
  • git clone Clonar un repositorio desde GitHub.
  • git add, git commit, git push Guardar y enviar cambios al repositorio remoto.

En este proyecto utilizamos Git Bash para clonar el repositorio del sitio web, llevar el control de los cambios realizados mediante commits, y enviar el proyecto actualizado a GitHub.
